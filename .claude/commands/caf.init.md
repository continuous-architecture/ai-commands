# CAF Init — Initialise a Continuous Architecture Project

## Context

You are an enterprise architect starting a new engagement using the **Continuous Architecture Framework (CAF)**.

Project context: $ARGUMENTS

---

## Objective

Bootstrap the folder structure, seed the key project files, and produce a `CLAUDE.md` that gives any AI assistant the context it needs to work effectively on this project.

---

## Steps

### 1. Create the project structure

Create the following folders and empty placeholder files:

```
project/
  principles.md          ← architecture principles (to be filled)
  requirements.md        ← functional & non-functional requirements
  stakeholders.md        ← key stakeholders
  data-model.md          ← domain model
  architecture-runway.md ← enablers backlog

experience-objectives/
  jtbd.md                ← jobs-to-be-done analysis
  personas.md            ← personas and experience maps

product/
  portfolio.md           ← product definitions

technology/
  fitness-functions.md   ← architectural fitness functions
  tech-principles.md     ← CAF technology principles assessment
  adrs/                  ← architecture decision records

operations/
  capabilities.md        ← VCAP capability assessment
  readiness.md           ← operational readiness

organisation/
  team-design.md         ← Team Topologies
  interaction-modes.md   ← inter-team contracts
  inverse-conway-audit.md ← Conway alignment

enterprise/
  operating-unit-map.md  ← OU hierarchy
  product-portfolio-map.md ← products per OU
  domain-map.md          ← bounded contexts
```

For each file, add a one-line comment: `<!-- Generated by caf.init — fill using the corresponding /caf.* command -->`.

### 2. Produce `project/principles.md`

Seed a principles file with the 4 CAF technology principles as a starting framework, plus 3 placeholder custom principles:

```markdown
# Architecture Principles — [Project name]

## CAF foundation principles

1. **Architect for evolutivity and modularity** — Modularity over monolith. Isolation enables independent delivery and resilience.
2. **Architect for scalability and resilience** — Design for failure. Systems must be responsive, resilient, elastic, and message-driven.
3. **Architect for continuous delivery and operability** — Architecture quality correlates with CI/CD performance. Build for operability from day one.
4. **Symbiotic coupling of machines and humans** — Augment humans with automation and AI; do not replace judgement with process.

## Custom principles (to be defined)

5. [Principle name] — [Statement]. Because: [rationale].
6. [Principle name] — [Statement]. Because: [rationale].
7. [Principle name] — [Statement]. Because: [rationale].
```

### 3. Produce `CLAUDE.md`

Create a `CLAUDE.md` at the project root. This file is read automatically by Claude Code at startup.

```markdown
# [Project name] — Continuous Architecture Project

## CAF context

This project uses the Continuous Architecture Framework (CAF).
All architecture work is organised across 6 views: Experience Objectives, Product, Technology, Operations, Organisation, Enterprise Decomposition.

## Available commands

Run `/caf.[command-name] [context]` to generate architecture artifacts.

Key sequences:
- Start: /caf.xo-jtbd → /caf.xo-personas
- Product: /caf.product-runway → /caf.practice-adr
- Technology: /caf.tech-principles → /caf.tech-fitness
- Organisation: /caf.org-team-design → /caf.org-interaction-modes → /caf.org-inverse-conway
- Enterprise: /caf.ed-operating-unit-map → /caf.ed-product-portfolio → /caf.ed-domain-map
- Review: /caf.review → /caf.report

## Project artifacts

All generated artifacts live in their view folder (experience-objectives/, product/, etc.).
ADRs live in technology/adrs/.

## Key constraints (fill these in)

- Programme name: [name]
- Scope: [brief description]
- Key stakeholders: [names/roles]
- Architecture principles: see project/principles.md
```

### 4. Print a startup summary

After creating all files, print a summary table:

| Folder | Files created | First command to run |
|---|---|---|
| project/ | principles.md, requirements.md, stakeholders.md, data-model.md, architecture-runway.md | Fill project/stakeholders.md, then /caf.xo-jtbd |
| experience-objectives/ | jtbd.md, personas.md | /caf.xo-jtbd |
| organisation/ | team-design.md, interaction-modes.md, inverse-conway-audit.md | /caf.org-team-design |
| enterprise/ | operating-unit-map.md, product-portfolio-map.md, domain-map.md | /caf.ed-operating-unit-map |
| technology/ | fitness-functions.md, tech-principles.md, adrs/ | /caf.tech-principles |
| operations/ | capabilities.md, readiness.md | /caf.ops-capabilities |

Finish with: "Project initialised. Start with `/caf.xo-jtbd [your programme context]` to define your experience objectives."
